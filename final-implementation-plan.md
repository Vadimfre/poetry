# Итоговый план реализации слайдера и календаря праздников

## Обзор

Задача: реализовать слайдер с картинками времен года на главной странице и календарь праздников, связанных со стихами.

## Составные части

1. **Бэкенд: сущности Holiday и PoemHoliday**
   - Добавление моделей в Prisma схему
   - Миграция базы данных
   - Создание модуля Holidays (контроллер, сервис, DTO)
   - Интеграция с существующим API

2. **Фронтенд: слайдер сезонов**
   - Замена компонента Hero на SeasonSlider
   - 4 слайда (зима, весна, лето, осень) с навигацией
   - Автопрокрутка и ручное управление
   - Связь с календарем (кнопка "Смотреть праздники")

3. **Фронтенд: календарь праздников**
   - Страница `/calendar` с фильтрацией по сезонам
   - Компонент SeasonCalendar для отображения праздников сеткой
   - Карточка праздника (HolidayCard) с переходом на детальную страницу

4. **Фронтенд: детальная страница праздника**
   - Динамический маршрут `/calendar/[slug]`
   - Отображение информации о празднике и связанных стихов
   - Навигация назад к календарю

5. **Админка: управление праздниками**
   - Расширение существующей админ-панели
   - Вкладка "Праздники" с CRUD операциями
   - Привязка стихов к праздникам

## Файлы с детальными инструкциями

В процессе планирования созданы следующие файлы:

1. `holiday-calendar-plan.md` – общий план, схема БД, бэкенд-модуль.
2. `holiday-api-integration.md` – интеграция API с фронтендом, хуки, типы.
3. `admin-holidays-plan.md` – расширение админки для управления праздниками.
4. `hero-slider-plan.md` – переработка Hero в слайдер сезонов.
5. `season-calendar-plan.md` – разработка компонента календаря для каждого сезона.
6. `calendar-api-integration.md` – связь календаря с API праздников.
7. `holiday-poems-display.md` – отображение стихов по выбранной дате.

## Порядок реализации

Рекомендуется выполнять шаги в следующем порядке:

### Этап 1: Бэкенд
1. Обновить схему Prisma (`schema.prisma`) – добавить модели Holiday и PoemHoliday.
2. Создать миграцию: `npx prisma migrate dev --name add_holidays`.
3. Сгенерировать Prisma Client.
4. Реализовать модуль Holidays (контроллер, сервис, DTO).
5. Добавить HolidaysModule в AppModule.
6. Протестировать endpoints через Postman или Swagger.

### Этап 2: Фронтенд – базовые типы и API
1. Создать типы для праздников (`holiday.types.ts`).
2. Создать API клиент (`holidays.api.ts`).
3. Создать React-хуки (`use-holidays.ts`).

### Этап 3: Слайдер
1. Создать компонент SeasonSlider (TSX + CSS).
2. Подготовить изображения для сезонов в `public/seasons/`.
3. Заменить Hero на SeasonSlider в `app/page.tsx`.
4. Протестировать слайдер.

### Этап 4: Календарь
1. Создать компонент SeasonCalendar (TSX + CSS).
2. Создать страницу `/calendar/page.tsx`.
3. Интегрировать с API праздников через хук useHolidays.
4. Реализовать фильтрацию по сезонам.

### Этап 5: Детальная страница праздника
1. Создать динамический маршрут `app/calendar/[slug]/page.tsx`.
2. Создать компонент HolidayDetail.
3. Реализовать получение праздника по slug (бэкенд endpoint).
4. Отобразить список стихов.

### Этап 6: Админка
1. Расширить админскую страницу: добавить вкладку "Праздники".
2. Реализовать CRUD интерфейс для праздников.
3. Добавить возможность привязки стихов.

### Этап 7: Интеграция и проверка
1. Проверить навигацию: слайдер → календарь → детальная страница.
2. Убедиться, что данные корректно загружаются.
3. Исправить возможные баги.

## Временные оценки

- Бэкенд: 2-3 часа
- Фронтенд (слайдер): 1-2 часа
- Фронтенд (календарь): 2-3 часа
- Детальная страница: 1-2 часа
- Админка: 1-2 часа
- Интеграция: 1-2 часа

Итого: ~8-14 часов.

## Примечания

- Изображения для сезонов нужно подготовить заранее (можно использовать placeholder).
- Для упрощения можно сначала реализовать базовый функционал без автопрокрутки и сложных анимаций.
- Связь стихов с праздниками (many-to-many) требует осторожности при удалении.

## Заключение

План готов к исполнению. Все необходимые инструкции и код приведены в приложенных файлах. Можно приступать к реализации, начиная с бэкенда.
